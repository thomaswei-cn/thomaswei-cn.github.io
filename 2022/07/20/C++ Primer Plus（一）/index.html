C++ Primer Plus（1）

第一~三章：变量类型

整型

整型变量分为无符号整型&有符号整型两大类。共10种。

  • char

    • 1字节
    • unsigned 0~255
    • signed -128~127
    • 默认情况下由C++实现决定是否有符号。
    • wchar_t:可表示系统使用的最大扩展字符集，其符号、长度由底层实现决定。
      • 对应的常量要加上前缀‘L’,L'p'表示wchar_t类型的字符p。
    • char16_t和char32_t，无符号，长度为16bit和32bit
      • 对应的常量要加上前缀‘u’/‘U’,u'p'表示char16_t类型的字符p；U'p'表示char32_t类型的字符p。

  • short

    • 2字节

  • int

    • 4字节
    • unsigned 0~65536
    • signed -32768-32767

  • long

    • 4字节

  • long long

    • 8字节

以上字节数&范围并不一定，因操作系统而异。各操作系统将相关信息定义在climits头文件中。

比如INT_MAX表示int的最大表示值；CHAR_BIT表示一个byte代表几个bit（一般为8）。

浮点型

浮点型一定有符号。且有着不同的存储/表示方式。

  • float：4字节
  • double：8字节
  • long double：不少于8字节

同样，字节数和所占空间因系统而异，具体可查阅cfloat头文件。

存储方式

由IEEE开发的IEEE标准提供了两种主要的浮点数格式：单精度(32位) 和双精度(64位)。数值以科学记数法的形式存储，每一个数都由三部分组成：符号、指数和小数。指数部分的位数说明了数值的可能大小程度，而小数部分的位数说明了精度。单精度格式中，指数长度为8位，而小数部分占了23位。因此，单精度数可以表示的最大值大约是3.40×10383.40×10^{38}3.40×1038，其中精度是6个十进制数字。

bool型

true&false两个值。

在转换为int类型时，true->1,false->0。

常量

字面常量

一般我们直接写出来的，像是123,0x456这种，称为字面常量。C++也会将他们视为基本类型中的一种。

整型

  • 不加后缀，一般视为int。
  • 后缀：
    • u:unsigned int
    • L:long;LL:long long
    • uLL:unsigned long long
    • …

浮点型

两种表示法：1.234和1.23e-3（后者表示1.23×10−31.23×10^{-3}1.23×10−3)

  • 不加后缀，一般视为double.
  • 后缀：
    • f/F:float
    • d/D:double

符号常量

  • const限定符：

    const int Month = 12表示声明一个int类型的常量Month，值为12，以后的程序中不允许修改该值。

    • 和#define用法类似，不过更推荐使用const，因为：
      1. 能够明确指定类型
      2. 使用C++作用域规则将常量限制在特定的文件或函数内。
      3. 可以定义更复杂的类型（数组、结构…)
    • 必须在声明的时候对const进行初始化。如果不这么做，则该常量的值是随机的，且无法修改。

类型转换

自动类型转换

自动类型转换发生在三种状态下：

  1. 初始化赋值
  2. 表达式计算（不同类型数据±*/…)
  3. 函数参数值传递

一般以不损失数据为原则，尽量把低级数据往高级数据转换（能表示的数越多越高级）。大多时候把整型提升为浮点型，因此又叫“整型提升”。

强制类型转换

普通类型（C风格）

(long) Month：并不改变Month的类型或值，而是新创建一个指定类型的值作为右值供使用。

C++强制转换

  • dynamic_cast<> ():

    （1）其他三种都是编译时完成的，dynamic_cast是运行时处理的，运行时要进行类型检查。

    （2）不能用于内置的基本数据类型的强制转换。

    （3）dynamic_cast转换如果成功的话返回的是指向类的指针或引用，转换失败的话则会返回NULL。

    • 对于类之间的转换：

      ![](D:\Blog\source_posts\C++ Primer Plus（一）\20180727195652929.png)

      1. 对于从子类到基类的指针转换 ,dynamic_cast 成功转换,没有什么运行异常,且达到预期结果，

      2. 从基类到子类的转换 , dynamic_cast 在转换时也没有报错,但是输出给 base2sub 是一个 nullptr ,说明dynamic_cast 在程序运行时对类型转换对*“运行期类型信息”（Runtime type information，RTTI）*进行了检查.

      这个检查主要来自虚函数(virtual function) 。

      在C++的面对对象思想中，虚函数起到了很关键的作用，当一个类中拥有至少一个虚函数，那么编译器就会构建出一个虚函数表(virtual method table)来指示这些函数的地址，假如继承该类的子类定义并实现了一个同名并具有同样函数签名（function siguature）的方法重写了基类中的方法，那么虚函数表会将该函数指向新的地址。此时多态性就体现出来了：当我们将基类的指针或引用指向子类的对象的时候，调用重写的虚函数时，就会顺着虚函数表找到对应子类的方法而非基类的方法。

      而dynamic_cast正是通过虚函数表找到对应函数的实现，才能确定从基类到子类的转换不会出现错误。

      但如果通过子类指针调用子类的特有实方法则会报错，因为原本的基类对象开辟的内存中并没有记录子类的这个函数的地址。

  • const_cast<> () :

    在C语言中，const限定符通常被用来限定变量，用于表示该变量的值不能被修改。

    而const_cast则正是用于强制去掉这种不能被修改的常数特性，但需要特别注意的是const_cast==不是用于去除变量的常量性==，而是去除指向常数对象的指针或引用的常量性，其去除常量性的对象必须为指针或引用。

    此时a的值打印出来仍为10，但如果将p&q解引用打印出来的值为20。

    规定：若原本指针指向的值为const，则不能用const_cast转换后的指针修改原值

  • static_cast<> () :

    （1）用于类层次结构中基类和派生类之间指针或引用的转换
    进行上行转换（把派生类的指针或引用转换成基类表示）是安全的
    进行下行转换（把基类的指针或引用转换为派生类表示），由于没有动态类 型检查，所以是不安全的
    （2）用于基本数据类型之间的转换，如把int转换成char。这种转换的安全也要开 发人员来保证
    （3）把空指针转换成目标类型的空指针
    （4）把任何类型的表达式转换为void类型

    static_cast:可以实现C++中内置基本数据类型之间的相互转换。

    如果涉及到类的话，static_cast只能在有相互联系的类型中进行相互转换

  • reinterpret_cast<type_id> () ：

    在C++语言中，reinterpret_cast主要有三种强制转换用途：

    1. 改变指针或引用的类型
    2. 将指针或引用转换为一个足够长度的整形
    3. 将整型转换为指针或引用类型

    用法：
    type-id必须是一个指针、引用。
    它可以做到指针<->整数

    在使用reinterpret_cast强制转换过程仅仅只是==比特位的拷贝==，因此在使用过程中需要特别谨慎！
