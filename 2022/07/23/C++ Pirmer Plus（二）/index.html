C++ Primer Plus（2）

第4~6章：复合类型&文件操作

复合类型指基于基本类型创建的其他类型变量。
（如指针就只能创建已知的几种基本类型的指针，而不能单独作为一个类型创建）

**（C++11新增）**列表初始化

以结构体初始化为例：

  • =可以省略
  • 不接受缩窄转换（如float转成int）
  • 只有定义变量的时候才能使用。

数组

声明与初始化

  • 因为数组在编译的时候分配空间，故arraySize必须在编译的时候有确定的值。
  • 如果不初始化，数组内的元素值不确定（内存中原来的值）。
  • 列表初始化如果没填满数组，则会把剩余元素设置为0。
    （如果直接={},那么所有元素均为0）

结构类型

结构体

位字段

共用体

关键字union，声明与结构体相同，但同一时间只能被当作一种类型。

枚举类型

关键字enum，被视为符号常量，从0开始赋值。

范围

  • max：（声明中的最大值为max），大于max的最小的2的幂-1
  • min：（声明中最小值为min），小于min的最小2的幂-1

强制转换

枚举类型可以在需要的时候自动向上转换为int，但只有在枚举范围内的int值能够被强制转换为枚举类型。

字符串

无论是C风格字符串还是C++中的string类，本质都是内存中一段char类型的数组，结尾用\0标识。

  • strlen()不包括\0，而size()包括\0。
  • 右值（字面量）返回的是指向第一个字母的指针（char*）
    不能将字符串字面常量等同于普通的字符串变量。这是因为编译器存储字面量的位置特殊以及是否进行了一些可能的操作（如复制多个副本）是不确定的。

指针

是什么：

  • 指针是程序数据在内存中的地址，而指针变量是用来保存这些地址的变量。

  • 指针的值（虚拟地址值）使用一个机器字的大小来存储,也就是说,对于一个机器字为w位（32/64）的电脑而言,它的虚拟地址空间是0→2w−10\to2^w−10→2w−1

指针变量：

  • 声明方式：

    • int* pointer = &a;

      • 编译时分配内存

      • 指向的内存区域位于“栈”中

    • int* pointer = new int;

      • 运行时分配内存

      • 指向的内存区域位于“堆”中

      • 使用完后需要用delete释放对应的内存区域

    • int* p1,p2;声明的是一个int类型的指针p1，和一个int类型的变量p2

    • 不能将整数直接赋值给指针变量，要int* pn = (int*) 0xB8000000;

包含信息：

  • 内存中某一处地址

  • 指向的地址中存储的数据类型

  • 其中void*类型指针不能直接解引用（*），要给定类型（强制转换）后才能解引用。

  • 运算：

    • 明确了类型的指针可以进行加减运算，每次+/-对应类型所占的内存大小

危险情况：

  • 若初始化的时候不赋初值，则指针变量会使用原本内存中的数据，导致指向未知的内存地址。

指针与数组：

  • 数组名相当于const指针，无法被修改

    • 对数组名取址（&）得到的是以整个数组为类型的指针

      如short tell[10];，则(&tell+2)将地址加20（bytes）。

    • sizeof(数组名)得到的是整个数组的字节数。

  • 指针变量因为存储了数据类型信息，故可以通过+/-运算表示数组。

创建方式：

  • 静态联编int tacos[10];

    • 数组长度在编译时设置。

  • 动态联编int* tacos = new int[size];其中size可以为变量。（动态数组）

    • 数组长度和空间在运行时设置。

    • 需要用delete[] tacos;释放内存。

一些需要填的坑

  • 智能指针
  • vector以及模板类的概念、
  • string类
  • 管理内存的四种方法：自动存储、静态存储、动态存储、线程存储（内存管理）
